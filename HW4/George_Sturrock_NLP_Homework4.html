Homework 3 - Question 1a¶

Question 1a asks us to calculate the edit distance between my given first name and nickname. The edit distance is a method used to quantify the similarity of two strings. The edit distance calculates similarity by measuring the number of changes, or edits, needed to transform one string to another. In this instance, we will calculate the number of edits required to transform my given first name to my nickname.

With a one syllable first name, nicknames were fairly rare for me. One that sticks out is "Pappageorgio". A friend coined this name after a character in National Lampoon's Vegas Vacation. There are obvious phonetic similarities between the two names, but what is the textual similarity as calculated by the edit distance method? The charts shown below show the matrix and straight line methods of manually calculating the edit distance. The highlighted green cells show the actual edit distance between "george" and "pappageorgio".

The matrix method is a rather complicated method which tracks the number of edit required to change the columnwise of the given name to the columnwise values of the nickname. For instance, the number of edits requried to convert "g" to "pap" is three. One edit would be to change "g" to "p". The second edit is to add "a". The third edit is to add "p". The straight line method simply tracks the columnwise changes between the two names. Both result in the same edit distance of seven.

Homework 3 - Question 1b¶

Question 1b from Homework 3 inquires about the percentage string match between my given name and nickname. This is a simple calculation where number of characters in common between the two names is divided by the total characters of the longest string. The edit distance calculated above can assist with this calculation. Subtracting the edit distance from the longest string returns the number of characters in common between the two strings. The function below calculates the percent string match.

In [37]:
def pctStringMatch(one, two, edist):
    if len(one) != 0 and len(two) != 0:
        long = (max([one, two], key=len))
        out = (len(long)-edist)/(len(long))
        return out
    else:
        return 0

Part of Speach Tagging Function¶

The "tagPOS" function implements NLTK, Pattern and Spacy part of speach (POS) taggers. The function inputs are the text to be tagged and the POS tagger to utilize. The return value is dependent upon the POS tagger utilized.

In [47]:
def tagPOS(text, tagger):
    if tagger=='pattern':
        print("Pattern POS Tagger")
        tagTokens = tag(text)
    elif tagger == 'spacy':
        print("Spacy POS Tagger")
        nlp = spacy.load('en')
        tagTokens = nlp(text)
    else:
        print("NLTK POS Tagger")
        tokens = nltk.word_tokenize(text)
        tagTokens = nltk.pos_tag(tokens)
    
    return tagTokens

The mode tag is adjective. The manual tag also identifies strong an adjective for further qualifies the token as a superlative adjective (JJS). In this instance, manual tagging appears to produce a superior result with a more descriptive and accurate tag.
