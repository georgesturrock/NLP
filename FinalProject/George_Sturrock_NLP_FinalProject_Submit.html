In the final project for Natural Language Processing, topic modelling of the IMDB movies reviews collected in prior assignments will be the base data set. The base movies for this study will be Rocky, Rocky 3 and Rocky 5. The movie review data is automatically scrapped from IMDB. The data will then be evaluated using Latent Dirichlet Allocation (LDA) to produce the topic models. This project will focus on different data engineering techniques to produce topic models. Using the raw review text, lemmatized review text, named entities, and hypernyms mixed with named entities will be explored for comparison. The raw review text model will use uncleansed text fed directly into the count vectorizer for LDA modeling. The remaining data engineering techniques will tokenize, part-of-speach tag, remove stop words, and remove punctuation prior to count vectorizing and LDA modeling. There are several challenges when topic modeling these three movies. Character and plot recylcling can make differentiating the three movies a difficult exercise. The results of each technique will be presented and summarized.

Create Additional Features for Topic Modeling¶

The code chunk shown below performs the steps needed to strip punctuation from the raw text reviews. The code begins by removing punctuation and stop words. Then, three different lists are populated to hold named entities, hypernyms, and lemmatized forms of the raw review text. The lists are then strung back together per row and added to the dfReviewDetails dataframe.

for dfr in dfReviewDetails.Review:
    tlist = []
    poslist = []
    nlist = []
    hlist = []
    t1 = tag(dfr, tokenize=True)
    for t, pos in t1:
        if t not in sw and t not in string.punctuation:
            if pos != "NNPS" and pos != "NNP":
                tlist.append(lemma(t))
                poslist.append(pos)
            else:
                nlist.append(t)
                tlist.append(t)
                poslist.append(pos)
            if pos.startswith("N") or pos.startswith("V"):
                if pos == "NNPS" or pos == "NNP":
                    hlist.append(t)
                else:
                    s = wordnet.synsets(t, pos = pos)
                    try:
                        sout = str(s[0].hypernyms(recursive=False, depth=None))[9:]
                        sout = sout.split('.')[0]
                        hlist.append(sout)
                    except Exception as e:
                        #print(t2, p2, e)

    lt = " ".join(l for l in tlist)
    nl = " ".join(n for n in nlist)
    hl = " ".join(h for h in hlist)
    lemmaList.append(lt)
    nerList.append(nl)
    hypernymList.append(hl)

def modelTopic2(listIn, minDf):
    countVectorizer = CountVectorizer(analyzer='word', min_df=minDf, stop_words='english', lowercase=True)
    matrix = countVectorizer.fit_transform(listIn)
    cvDense = matrix.todense()
    print("Matrix Sparseness:", round(((cvDense > 0).sum()/cvDense.size), 4))
    print("Count Vectorized Matrix Shape:", matrix.shape)
    clf = ldaGrid.fit(matrix)
    print("Log Likelihood:", round(clf.score(matrix), 4))
    print("Log Likelihood / Feature:", round((clf.score(matrix)/matrix.shape[1]), 4))
    print(clf.best_estimator_)
    #print(clf.best_params_)
    return clf, matrix, countVectorizer

LDA for Noun and Verb Hypernyms and Named Entities¶

WordNet is a lexical English database grouped into synsets. WordNet also contains hypernyms for most words in their database. A hypernym is a word with a broad meaning where more specific words fall underneath. This effectively creates a hierarchical structure for all words in the WordNet database. The use of hypernyms should create an actual topic-based scheme by which the LDA topic model will be created. Additionally, named entities will be included in the base of tokens used to create the LDA topic model.

Topic Model Summary¶

The named entity and hypernym plus named entity based LDA models produced the best results in terms of visual inspection, balance of topic models, and log likelihood per feature scoring. The log likelihood and log likelihood/feature scores can be manually influenced by the number of features in the shape of the count vectorized matrix. However, they do offer a numerical means of differentiating the different approaches. The named entity model produces the best results when considering visual inspection, topic balance, and numerical scoring. The obvious difference between Rocky (Apollo Creed), Rocky 3 (Clubber Lang), and Rocky 5 (Tommy Gunn) are the main protagonist characters. Named entity recognition does a decent job of creating topics based on Mr T's Clubber Lang character and Tommy Morrison's Tommy Gunn character. However, Apollo Creed is a main character in both Rocky and Rocky 3. The hypernym plus named entity approach likely produces results more applicable to a broader range of movies. By combining actor, character, and abstracted token data, this approach is capable of a layer of depth and insight which using named entities only cannot rival.
